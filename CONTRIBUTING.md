# Contributing to Arcane Studio

Arcane Studio currently focuses on two separate modules:

- Arcane Guard: safety inspection
- Arcane Themes: ricing/theme management

Arcane Guard is the current release focus.

Good contribution areas:

- reduce false positives
- add new safety rules
- improve report output
- add tests
- improve documentation
- test real AUR packages and dotfile repos

Rules should be explainable. A finding should tell the user what was found, why it matters, where it was found, and what to review next.
